1. 첫 화면 — 역할 선택
실행하면 세 역할이 세로로 표시된다. 원하는 역할을 클릭하면 해당 화면으로 이동한다.
 
첫 화면 — 민원인 · 당직원 · 처리자 (좌측 색 라인은 스마트안양 4색 브랜드)
· 좌상단 ‘당직등’ 로고 또는 각 화면의 ‘← 역할 선택’ 버튼을 누르면 첫 화면으로 돌아온다.
· 하단 정책환류 보기·설정 버튼으로 부가 화면에 접근한다.
 
2. 민원인 — 신고·접수
게시판·사진·메신저(카카오톡·라인·네이버톡톡) 중 방법을 고르고 내용을 입력한 뒤 ‘접수하기’를 누른다.
 
민원인 접수 화면 — 채널 선택 + 내용 입력

◆ 접수 결과 — AI 4단계 자동 응대
· 긴급: 119/112 안내와 함께 당직원에게 최우선 전달
· 단순 문의: AI가 즉시 안내문 제공
· 고충민원: 접수번호와 처리예정일 발급(아래 화면)
· 장난·주취: 필터링하되 ‘실제 긴급상황’ 재연결 버튼 제공
 
고충민원 접수 결과 — 접수번호·처리예정일 발급. 접수번호로 처리 상황 조회 가능
 
3. 당직원 — 검토·분류·이송
왼쪽은 전화·방문 직접 접수, 오른쪽 ‘검토 대기 큐’에는 민원인 접수분과 긴급 재연결 요청이 들어온다.
 
당직원 화면 — 큐에서 줄을 클릭하면 상세·SOP·재분류·이송 버튼이 펼쳐진다

◆ 조작 순서
[1] 큐에서 민원 줄을 클릭하여 상세 내용을 펼친다.
[2] AI 1차 분류·근거·대응절차(SOP)·비상연락처를 확인한다. 긴급 건은 119·112 등 연락처가 즉시 표시된다.
[3] AI 판단에 동의하지 않으면 ‘직접 재분류’ 드롭다운으로 분류를 바꾼다(사유 기록). 최종 판단은 사람이 한다.
[4] ‘✓ 확정 · 처리자에게 이송’ 을 눌러야 처리자(담당부서)에게 전달된다.
※ 중요: 민원인이 접수한 건은 당직원이 ‘확정·이송’을 눌러야 처리자 화면에 나타난다. 검토 없이 처리부서로 바로 넘어가지 않도록 한 의도적 단계이다.
 
4. 처리자 — 우선순위 요약카드
이송된 민원이 우선순위(P1~P4)순으로 정렬되어 표시된다. 각 카드에는 접수일시·핵심요지·분류근거·우선순위가 요약되어 출근 즉시 착수할 수 있다.
 
처리자 화면 — 우선순위 요약카드. 재분류된 건은 근거란에 사유가 함께 표시된다

◆ 주요 기능
· 처리 시작 / 완료 처리 버튼으로 상태를 갱신하면 타임라인에 자동 기록된다.
· 공문 초안 버튼으로 「민원 처리에 관한 법률」 근거의 부서 통보 공문 초안을 생성·복사한다.
· 아침 브리핑 전체 복사 로 대기 건을 일괄 복사해 메신저·공문에 붙여넣는다.
 
5. 정책환류 — 데이터 되먹임
응대 데이터가 쌓이면 반복지역·부서부하·재분류 이력·재연결 요청 등을 분석해 정책 제언을 자동 생성한다.
 
정책환류 화면 — 4단계 통계와 AI 정책 제언, JSON 내보내기
· 생성형 AI 심화 제언 버튼(API 키 필요)으로 행정 보고체 제언을 추가 생성한다.
· 환류 데이터 내보내기(JSON) 로 월간 보고·예산 편성 근거자료를 추출한다.
